## 1. Обзор продукта

**Название:** ИПН Инвестор  
**Платформа:** Telegram-бот + Web Mini App  
**Юрисдикция:** Республика Казахстан  
**Целевые пользователи:** Физические лица — резиденты РК, самостоятельно инвестирующие в ценные бумаги

### Краткое описание

ИПН Инвестор — сервис, который автоматически рассчитывает индивидуальный подоходный налог (ИПН) на инвестиционный доход физического лица, формирует декларацию по форме 240.00 и обеспечивает оплату налога в установленные законодательством сроки. Пользователь загружает брокерскую справку → получает готовую сумму к уплате → оплачивает в один тап.

## 2. Проблема и решение

### Проблема

Инвесторы-физические лица в РК обязаны самостоятельно рассчитывать и декларировать ИПН с инвестиционного дохода. На практике это создаёт три барьера:

| Барьер | Описание |
|--------|----------|
| **Сложность расчёта** | Необходимо применить FIFO-метод, учесть комиссии, конвертировать валюту по курсу НБ РК на каждую дату, применить льготы |
| **Незнание сроков** | Декларация — до 31 марта, уплата — до 10 апреля; большинство инвесторов не знают об этом |
| **Трудоёмкость** | Ручное заполнение формы 240.00 занимает несколько часов, требует специальных знаний |

Результат: налог не платится, штрафы, проблемы с КГД.

### Решение

Автоматизация всего цикла: загрузка справки из брокерского приложения → AI-парсинг → расчёт ИПН → генерация декларации 240.00 → оплата через Kaspi Pay.

> **Ключевое UX-обещание:** пользователь открывает бот в январе–марте и видит уже готовую сумму к уплате. Минимум действий — максимум уверенности.

## 3. Целевая аудитория

### Основной сегмент

Физические лица — резиденты РК, которые в течение года:
- Покупали и продавали акции (KASE, иностранные биржи)
- Получали дивиденды
- Торговали облигациями / использовали ИИС
- Совершали операции с криптовалютой

### Профиль пользователя

| Характеристика | Описание |
|----------------|----------|
| Возраст | 25–50 лет |
| Уровень | Начинающий или средний инвестор |
| Брокеры | Freedom Finance, Halyk Invest, Kaspi Invest, Interactive Brokers |
| Боль | «Я не знаю, сколько должен государству» |
| Мотивация | Спать спокойно, не бояться налоговых проверок |

### Что пользователь **не** является

- Профессиональным трейдером с сотнями сделок в день
- Юридическим лицом
- Нерезидентом РК

## 5. Пользовательский путь

### 5.1 Фоновый процесс (в течение года)

```
Янв–Дек    Пользователь периодически загружает брокерские отчёты
           ↓
           Система парсит, нормализует, сохраняет сделки
           ↓
           Фоновый движок считает текущий P&L и ориентировочный налог
           ↓
31 декабря  Система фиксирует финальный расчёт за год
           ↓
1 января    Push-уведомление: «Ваш налог за 20XX готов — X ₸»
```

### 5.2 Налоговый сезон (январь — апрель)

```
Шаг 1  Пользователь открывает бот
       → Видит: сводку налога, расшифровку по инструментам, два дедлайна

Шаг 2  Подача декларации 240.00
       → Бот показывает заполненную форму (ИИН, доход, налог)
       → Пользователь скачивает XML
       → Загружает на cabinet.egov.kz и подписывает ЭЦП
       → Отмечает в боте: «Подал» ✓

Шаг 3  Оплата налога
       → Кнопка «Оплатить X ₸»
       → Deep link в Kaspi.kz с предзаполненными реквизитами
       → Пользователь подтверждает платёж
       → Бот фиксирует: «Оплачено» ✓

Шаг 4  Напоминания на следующий год
       → Квартальные уведомления о накопленном P&L
       → Декабрь: «Загрузите финальный отчёт брокера»
```

### 5.3 Сценарий первого использования

```
1. /start → Приветствие, объяснение сервиса
2. Ввод ИИН → Проверка формата (12 цифр)
3. Выбор брокера → Freedom Finance / Halyk Invest / IBKR / другой
4. Загрузка файла (PDF / Excel / CSV)
5. AI-парсинг → Показ распознанных сделок для проверки
6. Подтверждение → Расчёт налога
7. Результат → Сумма к уплате с расшифровкой
```

## 6. Функциональные требования

### 6.1 Модуль: Импорт данных

**F-01 — Загрузка брокерских справок**

| Параметр | Значение |
|----------|---------|
| Форматы файлов | PDF, XLSX, XLS, CSV |
| Поддерживаемые брокеры (v1) | Freedom Finance Global, Halyk Invest, Interactive Brokers |
| Поддерживаемые брокеры (v2) | Jusan Invest, Kaspi Invest, BCC Invest, Robinhood |
| Максимальный размер файла | 20 МБ |
| Метод извлечения текста | pdfplumber (текстовые PDF), Tesseract OCR (сканы) |
| Парсинг | Claude API — универсальный AI-парсер |

**F-02 — Ручной ввод сделок**

Пользователь может добавить сделку вручную через форму:
- Тип (покупка / продажа / дивиденд / купон)
- Тикер / название инструмента
- Дата
- Цена за единицу
- Количество
- Комиссия (сумма)
- Валюта (KZT / USD / EUR / RUB)

**F-03 — Проверка и корректировка**

После парсинга пользователю показывается таблица распознанных операций. Пользователь может:
- Подтвердить все сделки
- Удалить некорректную строку
- Отредактировать любое поле
- Добавить пропущенную операцию

**F-04 — История загрузок**

Система хранит все загруженные файлы и распознанные сделки. При повторной загрузке того же периода — предупреждение о возможном дублировании.

---

### 6.2 Модуль: Налоговый движок

**F-05 — FIFO-расчёт прироста стоимости**

Расчёт ведётся по каждому инструменту отдельно.

```
Алгоритм FIFO:
1. Отсортировать покупки по дате (ранние — первые)
2. При продаже — списывать лоты с начала очереди
3. Для каждого списанного лота:
   Себестоимость лота = Кол-во × Цена покупки + Комиссия покупки (пропорционально)
4. Выручка от продажи = Кол-во × Цена продажи − Комиссия продажи
5. Доход по сделке = Выручка − Себестоимость
```

**F-06 — Учёт комиссий**

Комиссии брокера **уменьшают налоговую базу**:
- Комиссия покупки → увеличивает себестоимость
- Комиссия продажи → уменьшает выручку
- Учитываются: комиссия брокера, биржевой сбор, депозитарная комиссия

**F-07 — Конвертация валюты**

Все суммы конвертируются в тенге по **официальному курсу НБ РК на дату каждой операции**:

```
Себестоимость (KZT) = Цена_USD × Курс_НБ_РК(дата_покупки) × Кол-во
                    + Комиссия_USD × Курс_НБ_РК(дата_покупки)

Выручка (KZT) = Цена_USD × Курс_НБ_РК(дата_продажи) × Кол-во
              − Комиссия_USD × Курс_НБ_РК(дата_продажи)
```

Источник курсов: `https://api.nationalbank.kz/rss/rates_all.xml`

**F-08 — Зачёт убытков**

Убыток по одному инструменту уменьшает прибыль по другим инструментам в рамках **одного налогового года**. Перенос убытков на следующий год не предусмотрен (НК РК).

```
Налоговая база = max(0, Σ(Доход по всем инструментам))
```

**F-09 — Расчёт ИПН с дивидендов**

| Источник дивидендов | Ставка ИПН |
|--------------------|-----------|
| Казахстанские компании (KASE) | 5% |
| Иностранные компании | 10% |
| Примечание | Если налог удержан у источника — учитывается зачёт по СИДН |

**F-10 — Расчёт ИПН с купонного дохода**

| Инструмент | Ставка ИПН |
|-----------|-----------|
| Облигации (общий случай) | 10% |
| Облигации из официального списка KASE (ИИС) | 0% — льгота |

**F-11 — Льгота по сроку владения**

Акции, включённые в официальный список KASE, при владении **3 года и более** — освобождены от ИПН (ст. 331 НК РК).

```
Если тикер в списке KASE И дата_продажи − дата_покупки ≥ 1095 дней
   → ставка ИПН = 0%
```

**F-12 — Расчёт ИПН с криптовалюты**

Криптовалюта рассматривается как иное имущество. Доход = цена продажи − цена покупки (с учётом комиссий транзакций). Ставка ИПН = 10%. Источник данных: загрузка истории с биржи (Binance, Kraken, Bybit).

**F-13 — Итоговый расчёт ИПН**

```
ИПН_акции     = max(0, Σ P&L_акции) × 10%
ИПН_дивиденды = Σ Дивиденды_иностранные × 10%
              + Σ Дивиденды_казахстанские × 5%
ИПН_купоны    = Σ Купоны_облагаемые × 10%
ИПН_крипто    = max(0, Σ P&L_крипто) × 10%

ИПН_итого     = ИПН_акции + ИПН_дивиденды + ИПН_купоны + ИПН_крипто
```

### 6.3 Модуль: Декларация 240.00

**F-14 — Генерация XML**

Система формирует файл декларации формы 240.00 в формате XML, совместимом с Кабинетом налогоплательщика (cabinet.egov.kz).

Заполняемые поля:
- ИИН налогоплательщика
- Налоговый период (год)
- Доходы из зарубежных источников (раздел для иностранных акций)
- Доходы из источников в РК (казахстанские акции, дивиденды KASE)
- Сумма исчисленного ИПН
- Применённые льготы

**F-15 — Инструкция по подаче**

Бот предоставляет пошаговую инструкцию:
1. Перейдите на cabinet.egov.kz
2. Войдите через ЭЦП или ИИН + пароль
3. Раздел «Налоговая отчётность» → «Создать»
4. Выберите форму 240.00
5. Загрузите сформированный XML-файл
6. Проверьте данные
7. Подпишите ЭЦП и отправьте

> **Ограничение:** Бот не может подписать декларацию от имени пользователя — ЭЦП НУЦ РК выдаётся только физическому лицу лично. В v2 возможна интеграция с API КГД при наличии партнёрского соглашения.

**F-16 — Отслеживание статуса**

Пользователь отмечает в боте статус вручную:
- ☐ Декларация не подана
- ✓ Декларация подана (дата)
- ✓ Налог уплачен (дата, сумма)

## 7. Бизнес-логика и налоговые правила

### 7.1 Нормативная база

| Документ | Применение |
|---------|-----------|
| НК РК, ст. 320–321 | Ставки ИПН для физических лиц |
| НК РК, ст. 331 | Льгота при продаже акций (3 года, KASE) |
| НК РК, ст. 337 | Порядок исчисления ИПН с имущественного дохода |
| НК РК, ст. 358 | Порядок представления декларации по форме 240.00 |
| Приказ МФ РК № 166 | Форма и правила заполнения декларации 240.00 |

### 7.2 Сводная таблица ставок

| Инструмент | Ставка | Основание |
|-----------|--------|-----------|
| Прирост стоимости акций (иностранные или KASE < 3 лет) | 10% | ст. 320 НК РК |
| Прирост стоимости акций KASE (≥ 3 лет владения) | **0%** | ст. 331 НК РК |
| Дивиденды от компаний РК | 5% | ст. 320 НК РК |
| Дивиденды от иностранных компаний | 10% | ст. 320 НК РК |
| Купонный доход по облигациям (общий) | 10% | ст. 320 НК РК |
| Купонный доход по облигациям из списка KASE (ИИС) | **0%** | ст. 331 НК РК |
| Доход от операций с криптовалютой | 10% | ст. 337 НК РК |

### 7.3 Пример расчёта (FIFO + комиссии)

```
Покупка: 100 акций KCEL по 1 800 ₸, комиссия 540 ₸
Продажа: 100 акций KCEL по 2 200 ₸, комиссия 660 ₸

Себестоимость = 100 × 1 800 + 540               = 180 540 ₸
Выручка       = 100 × 2 200 − 660                = 219 340 ₸
──────────────────────────────────────────────────────────────
Доход         = 219 340 − 180 540                =  38 800 ₸
ИПН (10%)     = 38 800 × 0,10                    =   3 880 ₸

Без учёта комиссий налог был бы: (220 000 − 180 000) × 10% = 4 000 ₸
Экономия от учёта комиссий: 120 ₸
```

### 7.4 Пример расчёта с FIFO (несколько лотов)

```
Покупка 1: 150 акций HSBK по 72 ₸, комиссия 324 ₸  (2024-02-20)
Покупка 2: 100 акций HSBK по 85 ₸, комиссия 255 ₸  (2024-06-10)
Продажа:   200 акций HSBK по 95 ₸, комиссия 570 ₸  (2024-11-05)

FIFO:
  Лот 1: 150 шт из покупки от 20.02 → стоимость 150 × 72 + 324 = 11 124 ₸
  Лот 2:  50 шт из покупки от 10.06 → стоимость  50 × 85 + 127 =  4 377 ₸
  Итого себестоимость:                                            = 15 501 ₸

Выручка = 200 × 95 − 570                                         = 18 430 ₸
──────────────────────────────────────────────────────────────────────────────
Доход   = 18 430 − 15 501                                        =  2 929 ₸
ИПН     = 2 929 × 0,10                                           =    293 ₸
```

### 7.5 Зачёт убытков

```
KCEL:  +38 800 ₸
HSBK:  + 2 929 ₸
KCSA:  −10 860 ₸  (убыток)
──────────────────
Итого: +30 869 ₸
ИПН:  30 869 × 0,10 = 3 087 ₸  (вместо 4 173 ₸ без зачёта)
```

## 8. Интеграции

### 8.1 Брокерские отчёты (AI-парсинг)

**Метод:** Claude API (`claude-sonnet-4-6`) с промптом для извлечения сделок.

**Обрабатываемые форматы:**

| Брокер | Формат файла | Метод извлечения |
|--------|-------------|-----------------|
| Freedom Finance Global | PDF (текстовый) | pdfplumber → Claude API |
| Freedom Finance | XLSX | openpyxl → Claude API |
| Halyk Invest | PDF / XLSX | pdfplumber / openpyxl → Claude API |
| Interactive Brokers | CSV (Activity Statement) | прямой парсинг CSV → Claude API |
| Kaspi Invest | PDF | pdfplumber → Claude API |
| Любой другой | PDF / XLSX / CSV | OCR → Claude API |

**Промпт для Claude API:**

```
Извлеки все сделки из брокерской справки. Верни ТОЛЬКО JSON-массив.
Формат:
[{
  "date": "YYYY-MM-DD",
  "type": "buy|sell|dividend|coupon",
  "ticker": "ТИКЕР",
  "qty": число,
  "price": число,
  "commission": число,
  "currency": "KZT|USD|EUR",
  "amount": число  // для дивидендов и купонов
}]
Правила:
- Числа без пробелов, разделитель — точка
- Если валюта не указана → KZT
- Если комиссия отсутствует → 0
- Дата строго YYYY-MM-DD
```

**Постпроцессинг после парсинга:**
1. Валидация формата дат и чисел
2. Нормализация тикеров (верхний регистр, удаление суффиксов)
3. Дедупликация (проверка на уже существующие сделки)
4. Показ пользователю для подтверждения

### 8.2 Курсы валют — НБ РК

**Endpoint:** `https://api.nationalbank.kz/rss/rates_all.xml`

**Параметры запроса:**
```
GET https://api.nationalbank.kz/rss/rates_all.xml
    ?fdate=2024-01-15
    &lang=rus
```

**Логика получения курса:**
1. На каждую дату сделки в иностранной валюте — запрос в НБ РК API
2. Кешировать курсы в БД (курс не меняется задним числом)
3. Если API недоступен — использовать ближайший предыдущий известный курс

### 9.3 Схема базы данных (ключевые сущности)

```sql
-- Пользователи
users (
  id UUID PRIMARY KEY,
  telegram_id BIGINT UNIQUE,
  iin VARCHAR(12),
  full_name TEXT,
  created_at TIMESTAMP
)

-- Брокерские отчёты
broker_reports (

  period_end DATE,

  raw_text TEXT,

)

-- Сделки (нормализованные)
transactions (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users,
  report_id UUID REFERENCES broker_reports,
  date DATE NOT NULL,
  type VARCHAR(10),         -- 'buy', 'sell', 'dividend', 'coupon'
  ticker VARCHAR(20),
  qty DECIMAL(15,4),
  price DECIMAL(15,4),
  commission DECIMAL(15,4),
  currency VARCHAR(3),
  price_kzt DECIMAL(15,4),  -- конвертировано по курсу НБ РК
  commission_kzt DECIMAL(15,4),
  nbk_rate DECIMAL(10,4),   -- курс на дату сделки
  is_kase_listed BOOLEAN,
  created_at TIMESTAMP
)

-- Налоговые расчёты
tax_calculations (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users,
  tax_year INTEGER,
  gross_income_kzt DECIMAL(15,2),
  total_commissions_kzt DECIMAL(15,2),
  net_income_kzt DECIMAL(15,2),
  tax_base_kzt DECIMAL(15,2),
  dividend_income_kzt DECIMAL(15,2),
  tax_amount_kzt DECIMAL(15,2),
  declaration_xml TEXT,
  calculated_at TIMESTAMP,
  status VARCHAR(20)        -- 'draft', 'confirmed', 'filed', 'paid'
)

-- Курсы валют (кеш НБ РК)
exchange_rates (

  currency VARCHAR(3),

  PRIMARY KEY (date, currency)
)
```

### 9.4 AI-парсинг — поток данных

```
Файл (PDF/XLSX/CSV)
       ↓
[File Parser Service]
  1. Определить MIME-тип
  2. Если PDF → pdfplumber → извлечь текст
     Если скан → Tesseract OCR → текст
     Если XLSX → openpyxl → сериализовать в текст
     Если CSV  → прямое чтение
       ↓
[Claude API Request]
  model: claude-sonnet-4-6
  max_tokens: 2000
  prompt: <см. раздел 8.1>
       ↓
[JSON Response]
  Валидация схемы (Pydantic)
  Нормализация дат и чисел
  Определение биржи/листинга
       ↓
[NBK Rate Enrichment]
  Для каждой иностранной сделки:
    rate = get_nbk_rate(date, currency)
    price_kzt = price * rate
       ↓
[Deduplicate]
  Сравнить с существующими сделками пользователя
  Флаг возможных дублей
       ↓
[User Confirmation UI]
  Показать таблицу → пользователь подтверждает
       ↓
[Save to DB]
  Сохранить в transactions
  Запустить пересчёт tax_calculations
```

## 10. Безопасность и соответствие

### 10.1 Защита персональных данных

| Требование | Реализация |
|-----------|-----------|
| Шифрование ИИН | AES-256-GCM в БД; ИИН не хранится в открытом виде |
| Шифрование в транзите | TLS 1.3 для всех соединений |
| Минимизация данных | Брокерский файл удаляется после парсинга (хранить не более 30 дней) |
| Закон РК «О персональных данных» | Согласие при регистрации, политика конфиденциальности |
| Локализация данных | Данные хранятся на серверах в РК (требование закона) |

### 10.2 Юридические ограничения

> **Важно:** Сервис является **инструментом расчёта**, а не налоговым консультантом.

- При регистрации пользователь принимает оферту: «Сервис не является налоговым консультантом. Пользователь несёт ответственность за достоверность введённых данных и правильность поданной декларации»
- Расчёт основан на данных, предоставленных пользователем
- Все экраны с итоговыми суммами содержат дисклеймер: «Проверьте расчёт с квалифицированным налоговым консультантом»
- Рекомендация к специалисту при сложных случаях (СИДН, нерезидентство, налоговые споры)

### 10.3 Безопасность приложения

- Аутентификация через Telegram (OAuth, cryptographic verification)
- Rate limiting: не более 10 загрузок файлов в день на пользователя
- Антивирусная проверка загружаемых файлов (ClamAV)
- Логирование всех операций с персональными данными

## 12. Риски и митигации

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Ошибка расчёта → штраф пользователю | Средняя | Высокое | Дисклеймер в оферте; рекомендация к консультанту; возможность ручной корректировки |
| Изменение формы 240.00 / ставок ИПН | Высокая | Высокое | Модульный налоговый движок; подписка на обновления НК РК; быстрый цикл обновлений |
| Изменение формата отчёта брокера | Средняя | Среднее | AI-парсинг адаптируется автоматически; ручной ввод как запасной путь |
| Недоступность Claude API | Низкая | Среднее | Шаблонный ввод как fallback; retry с экспоненциальным backoff |
| Недоступность НБ РК API | Средняя | Среднее | Кеш курсов в БД; использование ближайшего известного курса |
| Нарушение закона о персональных данных | Низкая | Критическое | Юридическая экспертиза до запуска; хранение данных в РК; шифрование ИИН |
| Конкуренция со стороны банков/брокеров | Низкая (сейчас) | Высокое | Скорость выхода; UX-преимущество; независимость от брокера |
